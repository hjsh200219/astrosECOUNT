주요 변경사항

1. 전자결재 시스템 완전 재설계 (가장 중요!) ▼

기안자를 두 종류로 구분:

A. 기안자가 "프로그램(시스템)"인 경우 - 시스템이 자동으로 전자결재 생성

  • 계약(발주) 기안: 계약 등록 시 시스템 자동 기안
  • 구매(선적) 입력: 선적서류 수신 시 BL 일자 기준 서울외국환중개은행 매매기준율로 "실중량 × 단가 × 환율"을 자동 계산하여 전자결재 올림
  • 관세 기안: 관세 납부 시 시스템 자동 기안
  • 결재 흐름: 시스템(자동기안) → 담당자(확인) → 팀장(결재)

B. 기안자가 "담당자"인 경우 - 담당자가 직접 기안

  • 통관
  • 판매
  • 출고
  • 부대비 (적하보험 포함)
  • 외화지급결의서 (TT & PU)
  • 신용장개설품의서 (LC)
  • 결재 흐름: 담당자(기안) → 팀장(결재) 가 기본

전결 규칙:

  • 외화지급결의서 + 신용장개설품의서 = 대표이사 전결 (담당자→팀장→대표)
  • 그 외 모든 결재 = 팀장 전결 (시스템/담당자→담당자/팀장)
2. 수금관리 위치 변경 ▼

변경 내용: 업무 흐름도(TAB 2)에서 "수금관리"를 별도 단계로 빼지 않음

새로운 구조: 판매 확정 → [수금관리 실시간] → 출고

수금 상태(미수금/부분수금/수금완료)가 판매-출고 프로세스 내에서 실시간 모니터링되는 방식으로 변경

3. 비규격제품 중량차이 처리 시점 명확화 ▼

기존: 가중량과 실중량 차이 발생시 잡이익/잡손실로 처리

변경: 해당 계약의 판매가 모두 완료된 이후에만 잡이익/잡손실 정산 진행

이를 통해 중량차이에 대한 최종 정산을 정확하게 관리할 수 있음

4. 손익계산서 기간 선택 기능 ▼

기존: 월별 고정

변경: 기간을 자유롭게 선택하면 자동 산출

기능: 시작일~종료일 선택 → 해당 기간 매출/매출원가/판관비/영업이익 등 자동 계산

5. 기초데이터 관리(TAB 12) 수정 ▼

거래처 관리 변경:

  • "결제조건"과 "적하보험" 항목을 거래처 관리 테이블에서 삭제
  • 이 두 항목은 거래처가 아니라 계약관리에서 진행 (계약별로 결제조건/보험 다를 수 있음)

품목 관리 변경:

  • "기본 판매단가" 항목 삭제

창고 관리 변경:

  • "용량" (최대 수용량) 항목 삭제
  • "연락처" 항목을 유선번호 + FAX번호 두 필드로 분리
  • 출고증은 FAX 번호로 보내는 것이 디폴트 (이 점 명시)
✓ 전자결재 시스템 완전 재설계

⚡ 전자결재 시스템

시스템 자동기안(3종) + 담당자 기안(6종) 총 9가지 결재 유형

결재 유형 & 권한 체계

A. 시스템 자동 기안 (3종) ▼

정의: 시스템이 자동으로 전자결재를 생성하고, 담당자가 확인한 후 팀장이 결재하는 방식

결재 유형 결재 경로 설명
계약(발주) 기안 시스템 자동 → 담당자 확인 → 팀장 결재 계약 등록 시 시스템이 자동으로 결재 문서 생성
구매(선적) 입력 시스템 자동 → 담당자 확인 → 팀장 결재 선적서류 수신 시 BL 일자 기준 서울외국환중개은행 매매기준율로 "실중량 × 단가 × 환율"을 자동 계산하여 결재 올림
관세 기안 시스템 자동 → 담당자 확인 → 팀장 결재 관세 납부 시 시스템이 자동으로 결재 문서 생성
B. 담당자 기안 (6종) ▼

정의: 담당자가 직접 기안하여 팀장(또는 대표)이 결재하는 방식

결재 유형 결재 경로 설명 전결 규칙
통관 담당자 → 팀장 상품재고 전환, 회계 인식 팀장 전결
판매 담당자 → 팀장 규격/비규격 제품 구분 처리, 거래명세서 자동 생성 팀장 전결
출고 담당자 → 팀장 FAX 발송, 미트와치 자동 LIST UP 팀장 전결
부대비 (적하보험 포함) 담당자 → 팀장 창고비, 운송비, 적하보험 등록 팀장 전결
외화지급결의서 (TT & PU) 담당자 → 팀장 → 대표 TT 외화지급, PU 등록 대표 전결 ★
신용장개설품의서 (LC) 담당자 → 팀장 → 대표 신용장 개설 한도 관리 및 개설 대표 전결 ★
전결 규칙 정리
  • 팀장 전결: 계약 / 선적 / 관세 / 통관 / 판매 / 출고 / 부대비
  • 대표 전결: 외화지급결의서 / 신용장개설품의서
