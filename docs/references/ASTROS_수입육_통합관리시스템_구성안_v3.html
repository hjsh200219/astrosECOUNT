📋 개요 & v2→v3 핵심 변경점

ASTROS 수입육 통합관리시스템은 v2의 기본 자동화 기능을 대폭 확장한 v3.0으로 업그레이드됩니다. 메일함 전수 분석을 통해 발견된 모든 정형 업무 패턴에 대해 표준 템플릿과 자동화를 정의했으며, 비주얼 모니터링, 원클릭 처리, 자가진단 엔진 등 운영 효율성을 대폭 높이는 기능들이 추가됩니다.

v2 대비 v3 핵심 변경점

# 항목 v2 v3 상태
① 계약서 회신 없음 디지털 직인 자동 날인 + 회신 메일 초안까지 자동 작성 신규
② 메일 자동화 범위 주요 5개 전체 메일 분석 기반 15개 표준 템플릿. 모든 업무 메일에 자동 초안 생성 대폭 확대
③ 모니터링 텍스트 기반 비주얼 파이프라인 뷰: 전체 컨테이너를 시각적으로 실시간 추적. 단계별 색상 표시 신규
④ 원클릭 처리 출고만 통관요청·서류전달·원본요청·출고·검역증전달 등 6개 업무 원클릭 대폭 확대
⑤ 메일 표준화 개별 작성 15개 표준 템플릿 + 데이터 자동 매핑. 확인 후 발송만 신규
⑥ 수동 오버라이드 없음 모든 자동 데이터(환율/ETA/관세 등) 수동 입력·조정 가능 신규
⑦ 자가진단 없음 시스템 헬스 모니터링 + 오류 자동 감지 + 자동 수정 + 일일 리포트 신규
⑧ 비즈니스 대시보드 없음 수입추이·판매추이·마진분석·재고회전율 등 실시간 BI 대시보드 신규

모니터링 시스템 특징

① 파이프라인 뷰
칸반 보드 스타일의 시각적 배치. 각 단계에 컨테이너 카드가 시각적으로 배치되어 전체 프로세스를 한눈에 파악
② 컨테이너 카드
클릭하면 상세 정보 패널 + 액션 패널 열림. BL번호, 제품명, 현재 상태, 단계 소요일, 필요한 액션 등을 한눈에 표시
③ 알림 배지
⚠️ 노란 배지 = 담당자 확인 필요 (서류 미비, 원본 미수취 등)
🔴 빨간 배지 = 긴급 대응 필요 (ETA 변경, 통관 지연 등)
④ 필터
제품별, 공급업체별, 창고별, 상태별 필터로 관심 있는 컨테이너만 표시. 복합 필터 지원
⑤ 실시간 검색
계약번호, BL번호, 컨테이너번호로 즉시 검색 가능. 검색 결과 자동으로 카드 위치로 스크롤
⑥ 타임라인
각 컨테이너의 이동 경로와 각 단계별 소요일 표시. 예상 일정 vs 실제 일정 비교
⚠️ 담당자 개입 필요 알림
컨테이너가 담당자의 개입을 필요로 할 때, 파이프라인에서 해당 카드가 깜빡이는 알림을 표시합니다. 클릭하면 액션 패널이 열리며, 필요한 작업, 자동생성된 이메일 초안, 원클릭 발송 버튼이 표시됩니다. 특수한 상황에서는 수동 오버라이드 옵션도 제공됩니다.

🔘 원클릭 업무 처리 (Click-to-Process)

담당자가 개입해야 하는 모든 업무를 "선택 → 확인 → 클릭" 3단계로 단순화합니다. 각 워크플로우는 자동으로 필요한 메일, 서류, 알림을 생성하고, 담당자는 확인만 하면 됩니다.

1️⃣ 통관 요청 (Click-to-Clear) ▼
프로세스
① 파이프라인에서 "입항대기" 컨테이너 클릭
② "통관요청" 버튼 클릭
③ 시스템 자동 처리:
  - 관세법인 자동배정 (전지벌크→원스탑, 기타→정운)
  - 메일 초안 자동생성 (TPL-CUSTOMS-01)
  - 필요서류 체크리스트 표시
④ 담당자: 내용 확인 → "발송" 클릭
2️⃣ 선적서류 전달 (Click-to-Forward) ▼
프로세스
① 선적서류 수신 감지 시 알림카드 자동 표시
② "전달" 버튼 클릭
③ 시스템 자동 처리:
  - 삼현INT 메일 초안 생성 (TPL-SHIP-01)
  - 정운관세법인 메일 초안 생성 (TPL-SHIP-02)
  - 두 메일을 동시에 준비
④ 담당자: 확인 → "전체 발송" 클릭
3️⃣ 원본서류 요청 (Click-to-Request) ▼
프로세스
① ETA-10일 알림카드 자동 표시
② "요청" 버튼 클릭
③ 시스템 자동 처리:
  - 포르투갈어 요청 메일 생성 (TPL-DOCS-01)
  - 공급업체별 요청 내용 자동 구성
④ 담당자: 확인 → "발송" 클릭
4️⃣ 계약서 직인 회신 (Click-to-Sign) ▼
프로세스
① 계약서 수신 감지 시 알림카드 자동 표시
② "서명 & 회신" 버튼 클릭
③ 시스템 자동 처리:
  - PDF에 디지털 직인 자동 날인
  - 회신 메일 초안 생성 (TPL-CONTRACT-01)
  - 영문 회신 메일 자동 작성
④ 담당자: 계약 내용 검토 → "발송" 클릭
⑤ 직인 날인본과 회신 메일이 함께 발송됨
5️⃣ 출고 처리 (Click-to-Deliver) ▼
프로세스 (v2에서 확대)
① 재고 대시보드에서 제품 선택
② 판매 정보 입력 (판매처, 수량 등)
③ "출고" 버튼 클릭
④ 시스템 자동 처리:
  - 출고증 PDF 자동 생성
  - 창고 FAX 자동 발송 (TPL-DELIVERY-01)
  - 판매처 통보 메일 생성 (TPL-DELIVERY-02)
  - 재고 자동 차감
⑤ 담당자: 메일 확인 → "최종 발송" 클릭
6️⃣ 통관서류 전달 (Click-to-Distribute) ▼
프로세스
① 통관 완료 후 필증 수신 감지 시 알림카드 표시
② "서류 배포" 버튼 클릭
③ 시스템 자동 처리:
  - 해당 제품의 판매처 자동 식별
  - 확인증/필증 전달 메일 생성 (TPL-CUSTOMS-02)
  - 판매처별 개인화된 메일 작성
④ 담당자: 확인 → "발송" 클릭
✅ 원클릭 처리의 효과
• 각 업무 처리 시간 80% 단축 (평균 15분 → 3분)
• 메일 작성 오류 100% 제거 (표준 템플릿 자동 적용)
• 담당자는 확인만 하면 됨 (초안 작성 없음)
• 모든 내용이 데이터 기반으로 자동 채워짐

🏥 시스템 자가진단 & 자동수정 엔진

시스템이 24/7 스스로를 모니터링하고, 오류를 감지하고, 가능한 범위 내에서 자동으로 수정합니다. 모든 활동은 로그되어 감사 추적이 가능합니다.

3계층 모니터링 구조

Layer 1: 인프라 헬스 모니터링

⚙️ 실시간 모니터링 항목
• 웹서버 상태 (매 30초 ping 체크)
• DB 연결 상태 (커넥션 풀 모니터링)
• API 연동 상태 (MS Graph, 선사 API, 환율 API, FAX 서비스)
• 디스크/메모리 사용량 (알람 임계값 80%)
• → 이상 시: 자동 재시작 → 3회 실패 시 카카오톡 긴급 알림

Layer 2: 데이터 정합성 검증 (매일 AM 2:00)

🔍 매일 자동 검증 항목
• 계약 DB ↔ 선적 DB 매칭 검증 (orphan 선적 없는지)
• 재고 3단계 합계 검증 (미착+도착+상품+판매완료 = 총 계약수량)
• 환율 적용 검증 (BL date에 환율 누락 건)
• 통관 후 관세 미반영 건 검출
• 기초재고 ↔ 전월 기말재고 일치 검증
• OneDrive 폴더 ↔ DB 매칭 (누락 폴더 검출)

Layer 3: 업무 프로세스 감시

📌 실시간 업무 프로세스 감시
• 메일 자동감지 누락 (수신 메일 중 미처리 건)
• 선박 트래킹 미갱신 (24시간 이상 ETA 업데이트 없는 건)
• 서류 체크리스트 미완료 건 (입항 임박인데 서류 미비)
• 통관 지연 건 (입항 후 7일 이상 미통관)
• 출고 미처리 건 (판매 등록 후 3일 이상 미출고)
• → 지연/누락 발견 시: 담당자에게 알림 + 액션 제안

오류 유형 자동 수정 방법

• 모든 수정 이력 완전히 기록 (감사 추적 가능)

📈 실시간 비즈니스 대시보드 (BI)

수입육 사업의 핵심 지표를 실시간으로 시각화합니다. 경영진이 수입 추이, 판매 현황, 수익성을 한눈에 파악할 수 있습니다.

대시보드 섹션

📊 1. 수입 추이 (Import Trends) ▼
핵심 지표
• 월별 수입량 (MT) 차트: 돈육/계육 별도 + 총합
• 월별 수입금액 (USD/KRW) 차트
• 공급업체별 비중 (Aurora vs Vibra 파이차트)
• 제품별 비중 (전지/삼겹/목살/닭발/정육 등)
• 전년 동기 대비 증감률
📊 2. 판매 추이 (Sales Trends) ▼
핵심 지표
• 월별 판매량/판매금액 차트
• 판매처별 비중 (CJ/디케이통상/기타)
• 제품별 판매 순위 (가장 많이 팔리는 상품)
• 판매 속도 (재고 회전율)
• 판매 추세 예측 (다음 달 전망)
💰 3. 마진 분석 (Margin Analysis) ▼
핵심 지표
• 제품별 마진율 (판매단가 vs 원가+관세)
• 계약별 수익성 순위
• 환율 변동에 따른 마진 영향 분석
• 월별 마진 추이
• 가장 수익성 좋은 제품 TOP 5
📦 4. 재고 현황 (Inventory Status) ▼
핵심 지표
• 3단계 재고 금액 스냅샷 (미착/도착/상품)
• 재고 회전율 (일수)
• 창고별 재고 분포
• 유통기한 임박 재고 경고
• 평균 재고 보유 기간
✈️ 5. 물류 성과 (Logistics KPIs) ▼
핵심 지표
• 평균 리드타임:
  - 선적→입항 (해상운송 일수)
  - 입항→통관 (포트 처리 일수)
  - 통관→출고 (국내 처리 일수)
• 통관 소요일 평균
• 서류 완비율 (%)
• ETA 정확도 (예상 vs 실제 입항일 차이)
💵 6. 환율 모니터링 ▼
핵심 지표
• USD/KRW 30일 추이 차트
• 계약별 적용 환율 분포
• 환율 변동이 재고 평가에 미치는 영향
• 현재 환율 vs 30일 평균
• 환율 예측 (다음 주 전망)
✅ BI 대시보드의 가치
• 경영진은 매일 아침 대시보드로 전일 실적 확인
• 데이터 기반 의사결정으로 반응성 극대화
• 이상징후 즉시 감지 (마진율 악화, 재고 체증 등)
• 전략적 의사결정에 필요한 모든 정보 한눈에
• 데이터는 실시간 갱신 (수동 입력 불필요)

⚙️ 수동 오버라이드 & 데이터 관리

시스템이 자동으로 가져오는 모든 데이터는 필요 시 수동으로 입력/조정할 수 있습니다. "자동이 기본, 수동이 예외"의 원칙 하에 설계되었으며, 모든 수동 개입은 기록됩니다.

수동 오버라이드 카테고리

1️⃣ 환율 오버라이드 ▼
기능
• BL date 환율은 자동으로 API에서 조회됨
• 필요 시 수동으로 조정 가능
• 조정 사유는 필수로 기록 (감사 추적)
• 수동 입력 시 ⚡ 아이콘으로 시각적 표시
• 예: "선사 특별 할인율 반영" 등 사유 기록 가능
2️⃣ ETA/선박정보 오버라이드 ▼
기능
• 자동 트래킹으로 ETA가 실시간 갱신됨
• 선사 직접 확인 후 수동 ETA 입력 가능
• 수동 입력 후 자동 트래킹 재개할 때:
  - 자동값 우선 적용 또는
  - 수동값 유지 선택 가능
• 항상 차이 내역이 기록됨
3️⃣ 관세/원가 오버라이드 ▼
기능
• 필증 AI 파싱 결과는 자동으로 관세액 인식
• 파싱 오류 시 수동으로 수정 가능
• 추가 비용(부대비용, 수수료 등) 수동 입력 가능
• 원가 재계산 버튼으로 즉시 반영
• 수정 사유 기록 필수
4️⃣ 재고 수량 조정 ▼
기능
• 실사 후 차이 발견 시 수동 조정
• 조정 사유는 필수 기록
  - 예: "창고 실사 결과 수량 차이 발생"
• 조정 이력 전체 추적 가능
• 누가, 언제, 얼마나, 왜 조정했는지 모두 기록
• 부정행위 방지 (완전한 감사 추적)
5️⃣ 메일 초안 수정 ▼
기능
• 모든 자동 생성 메일 초안은 발송 전 수정 가능
• 수정할 때 "표준 vs 수정" 비교 표시
• 수정 패턴이 반복되면 템플릿 업데이트 제안
  - 예: "이 부분이 계속 수정되는데, 템플릿을 이렇게 바꿀까요?"
• 수정 내역 기록 → 향후 템플릿 개선에 반영
6️⃣ 계약/선적 정보 수동 입력 ▼
기능
• AI 파싱이 불가능한 복잡한 서류는 수동 입력 폼 제공
• 수동 입력도 동일한 DB에 저장
• 수동 입력 건은 🔧 아이콘으로 표시
• 입력 후 시스템이 상호 검증 (자동 데이터와 비교)
• 불일치 시 경고 표시
✅ 설계 원칙
"자동이 기본, 수동이 예외"
• 모든 수동 개입 항목에 시각적 마크 (⚡, 🔧 등)
• 수동 개입의 사유는 반드시 기록
• 모든 조정 이력은 감사 추적 가능
• 수동 수정 패턴 → 시스템 개선 피드백

🛡️ 장애대응 & 백업 전략

시스템이 장애 상황에 대비하여 다층적인 백업과 복구 전략을 갖추고 있습니다. v3에서는 자가진단 엔진과 확대된 메일 템플릿도 포함됩니다.

장애 대응 체계

✅ 예방 (Prevention)
• 자가진단 엔진 (Layer 1-3)으로 문제 사전 감지
• 매일 AM 2:00 데이터 정합성 자동 검증
• 인프라 헬스 매 30초 모니터링
• 이상 시 즉시 자동 재시작 (무중단 서비스 목표)
🔔 감지 (Detection)
• 3회 자동 재시작 실패 → 카카오톡 긴급 알림
• 데이터 정합성 점수 < 95점 → 아침 리포트에 경고
• API 연동 실패 → 대체 API로 자동 전환
• 담당자에게 즉시 알림 (우선도 표시)
🚨 대응 (Response)
• 인프라 장애: 클라우드 자동 스케일링 + 백업 서버 전환
• DB 장애: 실시간 복제 서버로 즉시 전환
• API 장애: 대체 데이터소스 또는 마지막 성공값 사용
• 메일 발송 장애: 발송 큐에 보관 → API 복구 시 자동 발송
• 담당자는 자가진단 리포트를 보고 필요한 액션 판단
📦 복구 (Recovery)
• 매시간 증분 백업 (자동)
• 매일 전체 백업 (자동)
• 매주 장기 보관 백업 (자동)
• 복구 목표 시간 (RTO): 1시간
• 데이터 손실 없음 (RPO: 15분)
