# 성숙도 프레임워크 (Maturity Framework)

> 12원칙을 4차원으로 그룹핑하고 가중 평균으로 종합 점수/등급을 산출한다.

## 4차원 그룹핑

### A. Documentation & Navigation — 가중치 30%

| # | 원칙 |
|---|------|
| P1 | Agent Entry Point |
| P2 | Map, Not Manual |
| P5 | Progressive Disclosure |
| P12 | Self-Documentation |

에이전트가 올바른 위치에서 올바른 정보를 얻는가. 모든 작업의 전제 조건이므로 최고 가중치.

### B. Enforcement & Consistency — 가중치 30%

| # | 원칙 |
|---|------|
| P3 | Invariant Enforcement |
| P4 | Convention Over Configuration |
| P10 | Reproducibility |

에이전트가 실수 없이 일관된 결과를 내는가. 품질 보장 메커니즘.

### C. Architecture & Knowledge — 가중치 20%

| # | 원칙 |
|---|------|
| P6 | Layered Architecture |
| P9 | Knowledge in Repo |
| P11 | Modularity |

복잡한 작업에서 올바른 판단을 내리는가. 변경 영향 범위 이해.

### D. Operations & Maintenance — 가중치 20%

| # | 원칙 |
|---|------|
| P7 | Garbage Collection |
| P8 | Observability |

시간이 지나도 에이전트가 효과적으로 작업할 수 있는가. 지속 가능성.

## 5등급 체계

| 등급 | 이름 | 종합 점수 | 의미 |
|------|------|----------|------|
| **L1** | None | 0-19 | 에이전트 협업 고려 없음 |
| **L2** | Basic | 20-39 | 최소한의 문서화 |
| **L3** | Structured | 40-59 | 체계적 구조, 부분 자동화 |
| **L4** | Optimized | 60-79 | 높은 자동화, 낮은 drift |
| **L5** | Autonomous | 80-100 | 에이전트가 독립적으로 작업 가능 |

## 등급별 상세

### L1 — None (0-19)

| 차원 | 상태 |
|------|------|
| A | README 없거나 오래됨. AGENTS.md 부재. |
| B | linter/formatter 미설정. CI 미구축. |
| C | 모듈 경계 불명확. 순환 의존성. |
| D | 환경 설정 구두 전달. CHANGELOG 없음. |

### L2 — Basic (20-39)

| 차원 | 상태 |
|------|------|
| A | 기본 README 존재. 불완전하거나 최신 아님. |
| B | linter 느슨. CI 커버리지 낮음. |
| C | 대략적 모듈 경계. 도메인 용어 일부 주석. |
| D | 기본 환경 스크립트. 변경 이력은 git log만. |

### L3 — Structured (40-59)

| 차원 | 상태 |
|------|------|
| A | 구조화된 문서 체계. AGENTS.md 초기 버전. |
| B | linter + formatter CI 강제. pre-commit hook. |
| C | 명시적 모듈 경계. 도메인 용어집 존재. |
| D | Docker 환경 재현. CHANGELOG 유지. |

### L4 — Optimized (60-79)

| 차원 | 상태 |
|------|------|
| A | 문서-코드 동기화 메커니즘. 포괄적 코드베이스 맵. |
| B | 엄격한 type-checking. 아키텍처 규칙 도구 검증. |
| C | 모듈별 독립 빌드/테스트. 풍부한 도메인 모델. |
| D | 원커맨드 환경 설정. 자동 릴리스. |

### L5 — Autonomous (80-100)

| 차원 | 상태 |
|------|------|
| A | Self-describing 코드베이스. 에이전트별 최적 안내. |
| B | Zero-tolerance. 패턴 준수율 95%+. |
| C | 완전한 모듈 독립성. 영향 자동 계산. |
| D | 완전 자동 프로비저닝. drift zero 수렴. |

## 점수 계산

### 1. 원칙 점수

각 원칙은 0-10 범위. `docs/harness/principles.md` 채점 기준 참조.

### 2. 차원 점수

```
DimA = (P1 + P2 + P5 + P12) / 4
DimB = (P3 + P4 + P10) / 3
DimC = (P6 + P9 + P11) / 3
DimD = (P7 + P8) / 2
```

### 3. 종합 점수

```
종합 = (DimA × 0.3 + DimB × 0.3 + DimC × 0.2 + DimD × 0.2) × 10
```

최대값: `(10×0.3 + 10×0.3 + 10×0.2 + 10×0.2) × 10 = 100`

### 4. 등급 결정

종합 점수를 5등급 체계에 매핑. **단, 어떤 차원이든 3.0 미만이면 해당 영역 에이전트 활용 불가로 판정.**

## 계산 예시

| 원칙 | 점수 |
|------|------|
| P1 | 8 |
| P2 | 7 |
| P3 | 8 |
| P4 | 7 |
| P5 | 6 |
| P6 | 8 |
| P7 | 7 |
| P8 | 9 |
| P9 | 7 |
| P10 | 8 |
| P11 | 7 |
| P12 | 7 |

```
DimA = (8 + 7 + 6 + 7) / 4 = 7.0
DimB = (8 + 7 + 8) / 3 = 7.67
DimC = (8 + 7 + 7) / 3 = 7.33
DimD = (7 + 9) / 2 = 8.0

종합 = (7.0×0.3 + 7.67×0.3 + 7.33×0.2 + 8.0×0.2) × 10
     = (2.10 + 2.30 + 1.47 + 1.60) × 10
     = 74.7 → L4 (Optimized)
```

## 등급 전환 체크리스트

### L1 → L2
- [ ] 루트에 최신 README.md
- [ ] 하나 이상의 linter/formatter 설정
- [ ] 빌드/실행 방법 문서화
- [ ] 기본 CI 파이프라인

### L2 → L3
- [ ] AGENTS.md 또는 CLAUDE.md 작성
- [ ] linter + formatter CI 강제
- [ ] 주요 코딩 패턴 문서화
- [ ] pre-commit hook 설정
- [ ] 환경 설정 자동화

### L3 → L4
- [ ] 문서-코드 동기화 메커니즘 (verify-docs 등)
- [ ] 엄격한 type-checking
- [ ] 아키텍처 규칙 도구 기반 검증
- [ ] 모듈별 독립 빌드/테스트
- [ ] 모든 차원 점수 최소 5.0 이상

### L4 → L5
- [ ] 모든 위반 자동 차단 (zero-tolerance)
- [ ] 패턴 준수율 95%+
- [ ] 완전한 모듈 독립성
- [ ] 모든 차원 점수 최소 7.0 이상

## 해석 가이드

| 등급 | 에이전트 활용 수준 |
|------|-------------------|
| L1 | 활용 불가. 기본 인프라 구축 필요. |
| L2 | 단순 작업만. 사람 검토 비용 높음. |
| L3 | 정형 작업 활용 가능. 반복 업무 30-50% 절감. |
| L4 | 대부분 일상 작업 독립 수행. 50-70% 절감. |
| L5 | 독립적 기여자. 개발 속도 2-3배 향상. |
